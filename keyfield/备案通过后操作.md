# 备案通过后操作指南

> 前提：gjinggg.art 备案审核通过，收到短信通知

---

## 1. Nginx 端口改回 80

```bash
sudo nano /etc/nginx/sites-available/blog-jing
```

把 `listen 8080;` 改回 `listen 80;`，保存退出后：

```bash
sudo nginx -t && sudo systemctl reload nginx
```

## 2. 申请 SSL 证书（HTTPS）

```bash
sudo certbot --nginx -d blog.gjinggg.art
```

按提示操作：
- 输入邮箱（用于证书过期提醒）
- 同意条款（Y）
- 是否接收邮件（N）
- HTTP 跳转 HTTPS（选 2，强制跳转）

Certbot 会自动修改 Nginx 配置并设置自动续期。

## 3. 更新 Umami 跟踪地址

本地修改 `src/layouts/BaseLayout.astro` 第 24 行：

```
改前：src="http://106.52.244.190:3000/script.js"
改后：src="https://blog.gjinggg.art:3000/script.js"
```

改完后重新部署：

```bash
./deploy/deploy.sh
```

## 4. Umami Nginx 反向代理（可选）

如果想让 Umami 也通过域名访问（不暴露端口），在 VPS 添加：

```bash
sudo nano /etc/nginx/sites-available/umami
```

粘贴：

```
server {
    listen 443 ssl;
    server_name umami.gjinggg.art;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

然后：

```bash
sudo ln -sf /etc/nginx/sites-available/umami /etc/nginx/sites-enabled/
sudo certbot --nginx -d umami.gjinggg.art
sudo nginx -t && sudo systemctl reload nginx
```
