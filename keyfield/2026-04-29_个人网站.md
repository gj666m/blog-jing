# 需求分析报告：个人网站

> 分析日期：2026-04-29
> 需求来源：用户个人需求
> 复杂度评级：中等

---

## 1. 需求概述

### 一句话目标
搭建一个综合型个人网站，以 Markdown 文件驱动内容，记录学习历程、工作经历与生活内容，支持搜索、分类、评论等基础功能，部署在自有 VPS 上。

### 需求背景
用户希望通过个人网站系统性地记录和展示自己的学习、工作和生活。目前没有现成的网站，需要从零搭建。用户有开发基础，主要依靠 Claude Code 辅助开发，希望在过程中学习代码。

### 需求来源
- 原始需求方：用户本人
- 沟通方式：需求文件 + 多轮对话
- 原始材料：`data/需求信息/个人网站.md`
- 参考网站：https://jiangmiemie.com/（基于 Docusaurus 的个人网站，视觉风格简洁，青绿色主题）

### 已确认的关键决策

| 项目 | 决策 |
|------|------|
| 网站定位 | 综合型（博客 + 主页 + 生活模块） |
| 目标读者 | 以自用为主，兼顾公开展示 |
| 内容类型 | 图文随笔 + 照片影像 + 兴趣推荐 |
| 技术背景 | 有开发基础，Claude Code 辅助，边做边学 |
| 内容管理 | Markdown 文件驱动 |
| 部署方式 | 自有 VPS |
| 视觉参考 | jiangmiemie.com（干净简洁、青绿色主题色） |
| 框架选型 | Astro（现代静态站点生成器，性能优秀，学习曲线友好） |
| 功能需求 | 搜索、标签分类、评论、访问统计、暗色模式 |

## 2. 业务规则

### 核心业务流程
用户编写 Markdown 文件 → 推送到 Git 仓库 → Astro 构建静态页面 → 部署到 VPS → 访客浏览内容

### 网站模块规划

参考 jiangmiemie.com 的模块设计，结合用户需求，规划以下模块：

| 模块 | 路由 | 内容 | 说明 |
|------|------|------|------|
| 首页 | `/` | 个人介绍 + 最近更新 | 网站入口，展示个人形象和最新动态 |
| 博文 | `/blog` | 技术文章、学习笔记、工作心得 | 核心内容模块，支持标签和分类 |
| 生活 | `/life` | 生活随笔、旅行记录、日常感悟 | 图文混合，可含照片集 |
| 书架 | `/bookshelf` | 读书笔记、资源推荐、收藏整理 | 兴趣推荐类内容的承载模块 |
| 关于 | `/about` | 个人简介、联系方式 | 简洁的自我介绍页 |

### 内容模型（Astro Content Collections）

```typescript
// 博文内容模型
interface BlogPost {
  title: string;          // 标题
  description: string;    // 摘要
  pubDate: Date;          // 发布日期
  updatedDate?: Date;     // 更新日期
  tags: string[];         // 标签
  category: string;       // 分类（技术/学习/工作）
  coverImage?: string;    // 封面图
  draft?: boolean;        // 是否草稿
}

// 生活内容模型
interface LifePost {
  title: string;
  description: string;
  pubDate: Date;
  tags: string[];         // 如：旅行、日常、感悟
  type: 'essay' | 'photo' | 'travel';  // 内容类型
  images?: string[];      // 照片集
  location?: string;      // 地点（旅行类）
}

// 书架内容模型
interface BookshelfItem {
  title: string;
  description: string;
  pubDate: Date;
  tags: string[];         // 如：编程、设计、随笔
  type: 'book' | 'tool' | 'resource';  // 类型：书籍/工具/资源
  rating?: number;        // 推荐指数（1-5）
  link?: string;          // 外部链接
}
```

### 数据流向
```
Markdown 文件（content/ 目录）
  → Astro Content Collections 解析
  → 按模块/标签/分类生成静态页面
  → Pagefind 构建搜索索引
  → 构建产物输出到 dist/
  → 部署到 VPS Nginx
  → 访客通过域名访问
```

### 边界条件与异常场景

| 场景 | 处理方式 | 优先级 |
|------|---------|--------|
| 文章包含大量图片 | 使用图片优化（Astro 内置），lazy loading | P1 |
| VPS 宕机 | 配置 Nginx 错误页面，建议配置自动重启 | P2 |
| 搜索索引过大 | Pagefind 支持分片加载，按需索引 | P2 |
| 评论服务不可用 | Giscus 基于 GitHub Discussions，稳定性高 | P2 |

## 3. 技术方案

### 技术选型

| 决策点 | 选择 | 理由 | 备选方案 |
|--------|------|------|---------|
| 框架 | Astro 5.x | 现代静态站点生成器，性能优秀，Markdown 原生支持，岛屿架构 | Docusaurus、Next.js |
| UI 组件 | React（局部交互） | Astro 支持多框架混用，交互部分用 React 实现 | Svelte、Vue |
| 样式方案 | Tailwind CSS 4.x | 原子化 CSS，快速实现参考站风格，响应式方便 | CSS Modules、UnoCSS |
| 全文搜索 | Pagefind | Astro 生态原生支持，无需外部服务，构建时生成索引 | Algolia（需申请） |
| 评论系统 | Giscus | 基于 GitHub Discussions，免费、无广告、支持 Markdown | Utterances（基于 Issues） |
| 访问统计 | Umami（自建） | 开源、隐私友好、轻量，可部署在用户 VPS 上 | Google Analytics、百度统计 |
| 暗色模式 | Astro + CSS 媒体查询 + 切换开关 | Tailwind 内置 dark mode 支持 | — |
| 内容管理 | Markdown 文件 + Astro Content Collections | 用户偏好，与 Claude Code 工作流契合 | CMS 后台 |
| 部署 | VPS + Nginx | 用户自有 VPS | Vercel、Netlify |
| 包管理器 | pnpm | 速度快、磁盘占用小 | npm、yarn |
| 版本管理 | Git | 标准实践 | — |

### 视觉设计要点（参考 jiangmiemie.com）

- **主题色**：墨绿色系（#059669），搭配中性灰
- **排版**：中文优先，正文 16px，行高 1.75，使用系统字体栈或思源黑体
- **布局**：最大宽度 1200px，内容区 800px，留白充足
- **卡片样式**：圆角 8px，轻阴影，hover 时轻微上浮
- **暗色模式**：深灰背景 #1a1a2e，保持墨绿主题色不变
- **响应式**：移动端优先适配，导航折叠为汉堡菜单

## 4. 任务分解

### Phase 规划

| Phase | 目标 | 涉及模块/文件 | 验证方式 | 依赖 |
|-------|------|-------------|---------|------|
| P1 | 项目初始化 + 基础布局 | astro.config、BaseLayout、Header、Footer、global.css、tailwind.config | 本地 `pnpm dev` 能看到带导航的基础页面 | 无 |
| P2 | 内容系统搭建 | content/config.ts、BlogPost/LifePost/BookshelfItem 模型、示例 Markdown 文件 | 写入测试 Markdown 能被正确解析和渲染 | P1 |
| P3 | 页面路由开发 | pages/index、pages/blog/index、pages/blog/[...slug]、pages/life/*、pages/bookshelf/*、pages/about | 各页面路由可访问，文章详情页正确渲染 | P2 |
| P4 | 搜索 + 标签分类 | Pagefind 集成、TagList 组件、标签页、分类筛选 | 搜索能找到文章，标签页能按标签过滤内容 | P3 |
| P5 | 交互功能 | ThemeToggle（暗色模式）、Comment（Giscus 评论）、PostCard 动效 | 暗色模式切换正常，评论能加载和提交 | P3 |
| P6 | 访问统计 | Umami 自建部署 + 网站集成 | Umami 后台能看到访问数据 | P1 |
| P7 | 视觉打磨 | 样式微调、响应式适配、图片优化、细节动效 | 移动端/桌面端显示正常，Lighthouse 性能 > 90 | P3-P5 |
| P8 | VPS 部署 | Nginx 配置、HTTPS（Let's Encrypt）、域名绑定、自动部署脚本 | 通过域名 HTTPS 访问网站正常 | P7 |

### 预估工作量
- Phase 1-2（基础搭建）：2-3 天
- Phase 3-5（核心功能）：3-4 天
- Phase 6-8（增强 + 部署）：2-3 天
- 总计：约 1.5-2 周（边学边做）
