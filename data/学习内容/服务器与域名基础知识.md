## 一、域名

### 什么是域名
域名就是网站的地址，比如 `gjinggg.art`。用户通过域名访问你的服务器，而不需要记住 IP 地址。

### 域名的组成
```
blog.gjinggg.art
 ├─    ├─     └─ 顶级域名（.art）
 │      └─ 二级域名 / 主域名（gjinggg）
 └─ 子域名（blog）
```

### 子域名
- 子域名可以**无限创建**，不额外收费
- 只需在 DNS 解析控制台添加 A 记录即可
- 例如：`blog.gjinggg.art`、`app.gjinggg.art`、`api.gjinggg.art`

### 域名注册
- 注册商：腾讯云、阿里云（国内）、Cloudflare（海外）
- 国内服务器必须**ICP 备案**才能解析域名，审核约 7-20 天
- 海外服务器无需备案

## 二、DNS 解析

### 什么是 DNS 解析
把域名翻译成服务器 IP 地址的过程。用户输入域名 → DNS 服务器查出对应 IP → 访问服务器。

### 常用记录类型

| 记录类型 | 作用 | 示例 |
|---------|------|------|
| A 记录 | 域名 → IP 地址 | `@` → `1.2.3.4` |
| CNAME 记录 | 域名 → 另一个域名 | `blog` → `gjinggg.art` |

### 基本配置
| 主机记录 | 记录类型 | 记录值 | 效果 |
|---------|---------|--------|------|
| `@` | A | VPS 公网 IP | `gjinggg.art` 可访问 |
| `www` | A | VPS 公网 IP | `www.gjinggg.art` 可访问 |
| `blog` | A | VPS 公网 IP | `blog.gjinggg.art` 可访问 |

## 三、HTTPS 与 SSL 证书

### 为什么需要 HTTPS
- 浏览器会标记 HTTP 站点为「不安全」
- 部分浏览器 API（如评论组件）强制要求 HTTPS
- 搜索引擎优先收录 HTTPS 站点

### Let's Encrypt + Certbot（推荐方案）
- **免费** SSL 证书
- **自动续期**，证书有效期 90 天，Certbot 自动续期
- 一条命令完成申请 + Nginx 配置

### 操作流程
```bash
# 1. 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 2. 申请证书（自动配置 Nginx）
sudo certbot --nginx -d gjinggg.art -d www.gjinggg.art

# 3. 添加子域名证书
sudo certbot --nginx -d blog.gjinggg.art
```

### 证书文件位置
- 证书：`/etc/letsencrypt/live/域名/fullchain.pem`
- 私钥：`/etc/letsencrypt/live/域名/privkey.pem`

### HTTP 自动跳转 HTTPS
Certbot 配置时会询问，选 `2` 即可自动设置。手动配置方式：
```nginx
server {
    listen 80;
    server_name gjinggg.art;
    return 301 https://$host$request_uri;
}
```

## 四、Nginx

### 什么是 Nginx
Web 服务器软件，负责接收用户的 HTTP/HTTPS 请求并返回网页内容。

### 基本操作
```bash
# 安装
sudo apt install nginx -y

# 启动
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx

# 检查配置是否正确
sudo nginx -t

# 重新加载配置（修改配置后执行）
sudo systemctl reload nginx

# 重启
sudo systemctl restart nginx
```

### 配置文件位置
- 主配置：`/etc/nginx/nginx.conf`
- 站点配置：`/etc/nginx/sites-enabled/default`

### 端口说明
| 端口 | 协议 | 说明 |
|------|------|------|
| 80 | HTTP | 默认网页访问端口 |
| 443 | HTTPS | 加密网页访问端口 |
| 其他（如 8000、6001） | 自定义 | 用户自己部署的项目 |

域名默认走 80/443 端口，其他端口的项目通过 `IP:端口` 访问，两者互不影响。

### 反向代理（了解）
当想用域名访问某个端口的项目时，通过 Nginx 反向代理实现：
```nginx
server {
    listen 443 ssl;
    server_name app.gjinggg.art;

    location / {
        proxy_pass http://127.0.0.1:8000;  # 转发到本地 8000 端口
    }
}
```

## 五、服务器端口与防火墙

### 检查端口监听
```bash
ss -tlnp | grep -E '80|443'
```

### 腾讯云防火墙
服务器本地端口开放 ≠ 外部可访问。需要在**腾讯云控制台**放行：

> 轻量应用服务器 → 实例 → 防火墙 → 添加规则

常见需要放行的端口：

| 端口 | 协议 | 用途 |
|------|------|------|
| 22 | TCP | SSH 远程登录 |
| 80 | TCP | HTTP 网页访问 |
| 443 | TCP | HTTPS 网页访问 |
| 8000+ | TCP | 自定义项目（按需放行） |

## 六、域名 + 服务器 + 网站的关系

```
用户输入域名（gjinggg.art）
    ↓
DNS 解析 → 找到服务器 IP
    ↓
请求到达服务器 443 端口（HTTPS）
    ↓
Nginx 接收请求 → 返回网页内容
    ↓
用户看到网站页面
```

静态站点部署流程：
1. 本地开发网站 → 构建生成静态文件（HTML/CSS/JS）
2. 将静态文件上传到服务器 Nginx 的网站目录
3. Nginx 提供文件服务 → 用户通过域名访问
