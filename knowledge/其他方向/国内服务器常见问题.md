# 国内服务器常见问题与解决方案

## Docker 镜像下载慢

### 问题
- Docker Hub 官方镜像在国内下载极慢或超时
- ghcr.io（GitHub Container Registry）在国内基本不可用

### 解决方案
1. 配置 Docker 镜像加速器

```bash
sudo nano /etc/docker/daemon.json
```

```json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.1ms.run"
  ]
}
```

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

2. 优先使用 Docker Hub 镜像替代 ghcr.io
   - 例如 Umami：用 `umamisoftware/umami`（Docker Hub）而非 `ghcr.io/umami-software/umami`
   - 腾讯云镜像加速对 ghcr.io 无效，只对 Docker Hub 有效

## Umami 自建部署

### Docker Compose 配置

```yaml
version: '3'
services:
  umami:
    image: umamisoftware/umami:postgresql-latest
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgresql://umami:umami@db:5432/umami
      HASH_SALT: <随机字符串>
    depends_on:
      - db
  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: umami
      POSTGRES_USER: umami
      POSTGRES_PASSWORD: umami
    volumes:
      - ./sql/schema.postgresql.sql:/docker-entrypoint-initdb.d/schema.postgresql.sql:ro
      - umami-db-data:/var/lib/postgresql/data
volumes:
  umami-db-data:
```

### 部署步骤

1. 在 VPS 上创建目录 `~/umami/`，放入 `docker-compose.yml`
2. `docker compose up -d` 启动
3. 防火墙放行对应端口（腾讯云轻量服务器在控制台操作）
4. 访问 `http://IP:3000` 进入后台
5. 默认账号 admin/umami，**首次登录立即改密码**
6. 添加网站 → 获取跟踪代码

### 集成到网站

在 BaseLayout 的 `<head>` 中加一行：

```html
<script defer src="http://IP:3000/script.js" data-website-id="你的website-id"></script>
```

域名配好后改为 https 地址。

## 防火墙端口管理（腾讯云轻量服务器）

- 不用命令行 `iptables`，在腾讯云控制台「防火墙」页面操作
- 需要放行的端口：
  - 80/443（HTTP/HTTPS，备案后）
  - 3000（Umami，可限制来源 IP）
  - 8080（临时访问端口，备案后可关闭）
  - 22（SSH）

## 服务器操作建议

- 用户对服务器操作不熟悉时，把每一步拆成单条命令逐步引导
- `cat > file << 'EOF'` 在终端粘贴时容易提前触发，用 `nano` 编辑器更可靠
- 多项目共享 VPS 时，备案前用不同端口区分站点
