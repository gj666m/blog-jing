# 个人网站项目：从零到部署完整流程

## 项目规划

### 需求分析先行
- 先写需求文档（参考 `keyfield/2026-04-29_个人网站.md`），明确功能范围和技术选型
- 按 Phase 拆分：P1 初始化 → P2 内容 → P3-P5 功能 → P6 统计 → P7 视觉 → P8 部署
- 每个 Phase 完成后立即构建验证

### 技术选型决策

| 需求 | 选择 | 理由 |
|------|------|------|
| 框架 | Astro 5（非 6） | 稳定成熟，与 Tailwind 4 集成文档完善 |
| 样式 | Tailwind CSS 4 | CSS-first 配置，无 JS config 文件 |
| 交互 | React 19（局部） | 只用于搜索/主题/评论等客户端交互 |
| 搜索 | Pagefind | 静态站点全文搜索，构建后生成索引 |
| 评论 | Giscus | 基于 GitHub Discussions，免费 |
| 统计 | Umami（自建） | 隐私友好，开源自部署 |
| 部署 | VPS + Nginx | 完全掌控，不依赖第三方托管 |

## Phase 执行顺序

### P1：项目初始化 + 基础布局
- 初始化 Astro 项目，安装依赖
- 创建 BaseLayout、Header、Footer
- 配置全局样式（主题色、暗色模式变量）
- 写 CLAUDE.md 项目常驻指令

### P2：Content Collections + 内容
- 定义内容 schema（blog/life/bookshelf）
- 用用户真实内容替换示例内容
- 区分"参考文档"和"博客文章"风格

### P3-P5：功能开发（可合并）
- 页面路由（列表页 + 详情页）
- 搜索（Pagefind 集成）
- 标签系统（跨集合聚合）
- 暗色模式切换
- 评论系统（Giscus）

### P6：访问统计
- Docker 部署 Umami + PostgreSQL
- 集成跟踪脚本到 BaseLayout

### P7：视觉打磨
- 动画、响应式网格、移动端优化
- 阅读时间、返回顶部、标签云

### P8：部署上线
- 域名 + DNS + Nginx + 部署脚本
- RSS 订阅
- 备案 + SSL（后续）

## 关键节奏

- **大改动（3+文件）先讨论方案再动手**
- **每个 Phase 完成后 `pnpm build` 验证**
- **Session 结束时更新项目文档**（进度总结 + AI_review）
- **文档体系**：CLAUDE.md（常驻）+ keyfield/（详细文档）+ knowledge/（知识沉淀）
